Deck Folders and Multi-Select

v0.17.0
May 2026

What's new

Three deck-manager quality-of-life upgrades that have been missing from MTGA since launch — finally landing in MTGA+.

Multi-select decks

Click the checkmark button at the bottom of your screen to toggle multi-select on and off. While it's on, clicking a deck tile adds it to (or removes it from) the selection instead of opening it — and a checkmark overlay shows you exactly what's about to be acted on.

Bulk delete

With one or more decks selected, a Delete action appears with a single confirm dialog showing the count and names. Clearing twenty draft leftovers is one click instead of twenty trips through the per-deck confirm modal. Nothing leaves your collection — same code path as MTGA's native delete, just batched.

Folders for organizing your collection

Make your own folders alongside MTGA's built-in My Decks, Starter Decks, and Example Decks sections. User folders pin to the top of the list and click to collapse / expand just like the native ones.

  • Create a folder with the new folder button at the top of the deck list.
  • Rename or delete a folder by right-clicking its header. Deleting a folder moves its decks back to My Decks — no decks are ever destroyed.
  • Move decks into a folder by selecting one or more decks in multi-select mode and choosing a folder from the bulk-action menu.

Folder assignments are stored locally in BepInEx\plugins\MTGAEnhancementSuite\settings.json, so they survive game updates and never touch your MTGA account.

Under the hood

  • Native UI co-opt. User folders use MTGA's own DeckFolderView prefab — same animations, same styling, same click-through behavior. We inject them at the top of the built-in folder list and route deck visibility through MTGA's existing per-folder filter, so there's no parallel rendering path to keep in sync.
  • Icon assets. The new selection / bulk-action UI uses a small set of PNG icons. The installer drops them on fresh installs, and existing v0.17+ clients auto-download them on first launch if missing — vector fallbacks render if the network is unavailable.
  • Custom caret. Folder-name input fields use a hand-rolled blinking caret because TMP's built-in caret doesn't render reliably on input fields constructed at runtime.
